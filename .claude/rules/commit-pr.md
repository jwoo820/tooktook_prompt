# 커밋 · PR 컨벤션

---

## 1. 커밋 메시지 형식

커밋 메시지는 `{type}: {내용}` 형식으로 작성합니다.

| 타입 | 사용 시점 |
|------|----------|
| `feat` | 새로운 기능 구현 |
| `add` | asset 혹은 라이브러리 추가 |
| `chore` | 잡일 (버전 코드 수정, 패키지 구조 변경, 파일 이동, 변수명·reformat 등) |
| `fix` | 버그·오류 해결 |
| `docs` | README, WIKI 등 문서 개정 |
| `refactor` | 내부 로직 변경 없이 코드 구조를 개선하는 리팩토링 |

### 주의사항

- **허용된 타입만 사용**: 위 6가지 외에 `test`, `style`, `build` 등을 임의로 사용하지 않습니다. 애매하면 반드시 사용자에게 먼저 묻습니다.
- **`CLAUDE.md` 및 `.claude/` 폴더는 커밋하지 않습니다.** 개인 AI 작업 가이드이므로 `.gitignore` 처리는 하지 않고 수동으로 제외합니다. `git add .` / `git add -A` 사용 시 의도치 않게 포함될 수 있으니 주의합니다.

### 예시

```
feat: 검색 결과 페이지네이션 구현
fix: 팔로우 버튼 중복 탭 시 상태 꼬임 수정
chore: SearchFeature 유닛 테스트 설정 개선
refactor: HomeCore switch 케이스 MARK 섹션 분리
docs: rules에 커밋 컨벤션 추가
```

## 2. PR 전 브랜치 전략 (Rebase & Merge)

> ⚠️ **Git rebase는 히스토리를 재작성하는 예민한 작업입니다.** AI가 rebase 관련 명령어를 실행할 때는 **반드시 사용자에게 먼저 확인을 받고, 단계별로 진행합니다.**

### 지향하는 Merge 방식

```
feature/mvp/main ──●──●──●──●  (Merge commit으로 합병 이력이 가시적으로 남음 ✅)
                        ↑
feature/mvp/KAN-99 ──●──●
```

작업 브랜치를 main에 합병할 때 **Merge commit**을 남겨서 어떤 브랜치가 언제 합병되었는지 그래프에서 가시적으로 확인할 수 있게 한다.

### PR 전 필수 절차 (Rebase)

```bash
# Step 1. 내 작업 브랜치에서 main 기준으로 rebase
git rebase feature/mvp/main

# Step 2. 충돌이 발생했다면 해결 후 빌드 확인
git add .
git rebase --continue
# 이후 Xcode에서 빌드 확인

# Step 3. 원격 브랜치에 push
git push
# 이미 원격에 올라가 있어 push가 거부되는 경우에만 -f 사용
# ⚠️ 강제 push는 원격 히스토리를 덮어씁니다. 로컬 내용이 맞는지 반드시 확인 후 실행!
git push -f
```

> `-f` (강제 push)는 로컬과 원격의 상태가 달라 일반 push가 거부될 때만 사용합니다. 평소에는 지양하세요 — 작업 내용을 날릴 수 있습니다.

## 3. PR 본문 템플릿

```md
## [#이슈번호] 타입 : 작업 요약

## 무엇에 관한 PR 인가요? 🙋
- (문제/배경을 1~3줄로 설명)

## 어떤 것을 작업하셨나요? 🛠
- 작업1
- 작업2

## 🌱 PR Point
- 리뷰어가 꼭 봐야 할 변경 포인트 1
- 리뷰어가 꼭 봐야 할 변경 포인트 2

## 관련된 문서 📄
- API 문서 / 기획 문서 / 디자인 링크
```

### 작성 규칙

- 제목은 `[#이슈번호] {TYPE} : {핵심 변경}` 형식을 사용합니다.
- `무엇에 관한 PR`에는 왜 이 변경이 필요한지 배경을 적습니다.
- `어떤 것을 작업`에는 실제 변경 파일/기능 단위를 bullet로 적습니다.
- `PR Point`에는 리스크가 있거나 확인이 필요한 항목만 간결하게 적습니다.
- API 응답 스펙·분기 조건·테스트 방법이 바뀌었으면 반드시 `PR Point`에 명시합니다.
- PR 본문은 과장 없이 사실 기반으로 작성하고, 미완료 항목은 `TODO`로 명확히 분리합니다.
